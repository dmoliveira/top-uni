Custom domain setup for GitHub Pages

  1. Buy or choose your domain/subdomain.
  2. In GitHub repo settings, open Pages and enter the custom domain.
  3. Create a docs/CNAME file containing only your domain, for example cs-top.example.com.
  4. Update your DNS:
    • Subdomain: add a CNAME record pointing to dmoliveira.github.io.
    • Apex/root domain: use GitHub Pages A/AAAA records from GitHub’s official docs.
  5. Wait for DNS to propagate, then enable HTTPS in GitHub Pages.

Helpful commands

printf 'cs-top.example.com\n' > docs/CNAME
git add docs/CNAME
git commit -m "Add custom domain configuration"

Notes

  • Do not commit a placeholder CNAME unless you really own and want to use that domain.
  • This project already publishes from /docs, so no extra build step is needed.
